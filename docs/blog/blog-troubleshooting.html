    有道云笔记导出失败怎么办？登录、权限和资源排查清单

    7分钟 林青 · 问题排查 2026年7月21日

    批量导出失败时，不要先怀疑所有笔记都坏了。多数问题都集中在登录状态、Chrome 站点权限、导出范围过大、网络中断或少数特殊资源读取失败。

    下面这份清单按发生概率排序。建议先用一个小文件夹复现，再逐步扩大范围，这样更容易判断是环境问题还是某类笔记内容问题。

    先确认登录状态

    有道云笔记导出助手复用浏览器里 note.youdao.com 的登录状态，不需要你手动复制 Cookie。如果扩展提示未登录，先在同一个 Chrome 用户配置中打开有道云笔记网页端并刷新。

    • 确认不是在另一个 Chrome 个人资料里登录。
    • 确认无痕窗口没有阻止扩展读取站点状态。
    • 如果网页端已经过期，重新登录后再回到扩展弹窗。
    • 长时间导出前，避免中途退出有道账号。

    检查扩展站点权限

    如果笔记本树加载不到，常见原因是扩展没有访问有道域名的权限。打开 chrome://extensions/，进入扩展详情页，检查站点访问权限是否允许访问有道云笔记相关页面。

    如果你刚刚修改过权限，建议重新打开扩展弹窗，再点击“加载笔记本”。

    缩小导出范围

    第一次导出全量笔记时，图片、白板和历史资料较多的账号可能需要更长时间。可以先选择一个 10 到 30 篇笔记的小文件夹试导出，确认 ZIP 能正常生成，再导出更大的笔记本。

    现象建议处理
    一直停在加载笔记本重新登录网页端，检查扩展站点权限。
    导出到一半变慢等待后台任务处理大图、白板或特殊格式。
    某几篇失败先下载成功部分，再单独导出失败笔记所在文件夹。
    ZIP 没有出现检查浏览器下载权限、磁盘空间和下载目录。

    图片无法显示怎么办

    导出后的 Markdown 通过相对路径引用同级 assets/ 目录。图片无法显示时，先确认 Markdown 文件和 assets 文件夹没有被分开放置。

    1. 解压 ZIP 后不要只移动单个 .md 文件。
    2. 打开 Markdown，检查图片链接是否指向 assets/xxx。
    3. 确认资源文件确实存在于同级 assets 目录。
    4. 如果只有少数图片缺失，回到原笔记检查图片是否仍能在有道网页端打开。

    特殊格式失败怎么判断

    表格、思维导图、Draw.io 和 Excalidraw 这类内容会经过专门转换。若普通笔记都成功，只有特殊格式失败，可以优先抽查这类笔记。

    • 表格会尽量转换为 Markdown table。
    • 思维导图会展开为 Markdown 层级列表。
    • Draw.io 和 Excalidraw 会生成本地 SVG 资源并在 Markdown 中引用。
    • 图片文件会生成一个 Markdown 包装文件，并把图片保存到 assets。

    更详细的格式说明可以看 有道特殊格式处理说明。

    导出完成后立刻做一次验收

    不要把 ZIP 下载完成等同于备份完成。建议马上解压，随机打开 5 到 10 篇笔记，重点检查图片、表格、目录层级和你最重要的资料。

    什么时候需要反馈问题

    如果你已经确认登录、权限、网络和磁盘空间都正常，但稳定复现同一类失败，可以在 GitHub Issue 中提供扩展版本、浏览器版本、失败现象、是否含图片或特殊格式，以及是否能用小文件夹复现。不要上传包含隐私的原始笔记内容。
