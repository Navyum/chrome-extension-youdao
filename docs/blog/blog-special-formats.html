    表格、思维导图、白板怎么导出？有道特殊格式处理说明

    7分钟 开发笔记 · 格式转换 2026年7月15日

    有道云笔记里的特殊格式，导出时最容易丢信息。比如表格不是普通 HTML 表格，思维导图不是 Markdown 列表，白板也不是一张简单图片。导出助手的策略是：最终 ZIP 只保留 Markdown 和 Markdown 依赖资源。

    最终 ZIP 长什么样

    无论原始文件是什么格式，导出结果都会被整理为统一结构：

    Folder/
  Note.md
  Table.md
  Mindmap.md
  Board.md
  assets/
    Note_001.png
    Board.svg

    原始 .drawio、.mindmap、.lxtable 和 .excalidraw 文件不会被保留。这样做的好处是迁移后结构更干净，Markdown 工具可以直接打开。

    格式转换对照表

    有道来源导出结果说明
    标准笔记Markdown正文转换为 .md 文件。
    Markdown 笔记Markdown + assets保留 Markdown，并本地化图片资源。
    图片文件Markdown 包装 + 本地图片生成 .md 文件链接 assets 中的图片。
    .lxtable 表格Markdown 表格解析单元格数据，输出 Markdown table。
    .mindmap 思维导图Markdown 大纲节点树转换为缩进列表。
    Draw.ioMarkdown + SVGSVG 保存到 assets，Markdown 中引用。
    ExcalidrawMarkdown + SVG白板元素渲染为 SVG 资源。

    为什么不保留原始特殊文件

    原始特殊文件通常依赖有道云笔记自己的解析逻辑。即使导出，也很难在其他笔记工具里直接打开。把内容转换成 Markdown 或 SVG，虽然牺牲了一部分可编辑性，但换来了长期可读性和跨工具兼容性。

    表格转换

    有道表格会被转换为 Markdown 表格。适合保存清单、简单台账、对比表等内容。复杂合并单元格或高度格式化的表格，迁移后建议人工抽查。

    思维导图转换

    思维导图的节点会转换为 Markdown 大纲列表。这样可以直接在 Obsidian、Logseq 或其他支持列表折叠的工具中继续整理。

    Draw.io 与 Excalidraw

    白板类内容会输出为 Markdown 文件，并把对应 SVG 保存到 assets 目录。Markdown 中使用相对链接引用 SVG，迁移后即使离线也能查看图形。

    导出后怎么检查

    • 打开几个表格笔记，确认行列内容没有明显错位。
    • 检查思维导图层级，确认主要节点顺序正确。
    • 打开白板类 Markdown，确认 SVG 链接能显示。
    • 在目标工具里测试图片和 SVG 的相对路径。

    写在最后

    特殊格式导出的目标不是复刻有道云笔记编辑器，而是尽量把内容转为可长期保存、可迁移、可审计的文件。对于重要笔记，导出后做一次抽查仍然是必要步骤。
