    有道云笔记怎么批量导出 Markdown？零命令行方案

    8分钟 小麦 · Markdown 工作流 2026年7月15日

    如果你想把有道云笔记迁移到 Obsidian、Typora、VS Code 或其他本地 Markdown 工具，最省事的方式是直接导出为 .md 文件和本地 assets 资源目录。

    传统做法往往要逐篇复制，或者用脚本配置 Cookie。Chrome 扩展方案把这些步骤收进浏览器界面里：你登录有道云笔记，选择要导出的笔记本，然后等待 ZIP 下载。

    导出前准备

    • 使用 Chrome 或 Chromium 系浏览器。
    • 在浏览器中打开 note.youdao.com 并登录你的有道云笔记账号。
    • 确认扩展对 youdao.com 的站点访问权限已开启。
    • 给浏览器预留足够磁盘空间，尤其是图片较多的笔记本。

    Step 1：打开扩展并加载笔记本

    点击浏览器工具栏中的有道云笔记导出助手图标。如果还没有登录，扩展会提示你先打开有道云笔记登录。登录完成后回到弹窗，点击“加载笔记本”。

    Step 2：选择导出范围

    扩展会展示你的笔记本树。你可以全选，也可以只勾选某个笔记本或文件夹。建议第一次迁移时先选一个小文件夹做试导出，确认结果符合预期后再导出全部内容。

    Step 3：开始导出

    点击“开始导出”后，扩展会在后台逐篇读取笔记、下载引用资源、转换 Markdown，并持续更新进度。关闭弹窗不会停止任务，重新打开后可以继续查看状态。

    导出过程中不要退出浏览器账号。如果登录状态失效，扩展可能无法继续读取后续笔记。

    Step 4：下载 ZIP 并解压

    导出完成后，浏览器会下载一个 ZIP 文件。解压后你会看到类似结构：

    Notebook/
  Project/
    Meeting Notes.md
    Mindmap.md
    assets/
      Meeting Notes_001.png
      Board.svg

    Markdown 文件可以直接打开；图片和 SVG 等资源保存在同级 assets 目录中，文件内链接使用本地相对路径。

    导出质量检查

    建议导出后抽查这些点：

    检查项判断方式
    目录结构解压后文件夹是否符合原笔记本层级。
    图片资源随机打开含图片的 Markdown，确认图片能显示。
    表格打开原本的有道表格笔记，确认已转为 Markdown 表格。
    思维导图确认层级是否按 Markdown 列表展开。
    白板确认 Markdown 中是否链接了本地 SVG 文件。

    常见问题

    为什么只导出 Markdown？

    Markdown 是最适合长期保存和跨工具迁移的通用文本格式。当前项目刻意把输出收敛为 Markdown 和 assets，避免生成一堆难维护的中间格式。

    导出会影响原笔记吗？

    不会。扩展只读取数据并在本地转换，不会修改或删除有道云笔记中的原始内容。

    单篇失败怎么办？

    扩展会记录失败项，单篇失败不会阻塞整个导出流程。你可以根据错误提示重新登录、检查网络或缩小范围后再试。

    下一步迁移

    如果你的目标是 Obsidian，可以直接把解压后的目录作为 Vault 打开。完整操作可以继续看 有道云笔记迁移到 Obsidian 指南。
